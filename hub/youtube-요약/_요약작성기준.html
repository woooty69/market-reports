유튜브 요약 작성 기준 (모든 신규 링크에 적용)

새 유튜브 링크를 요약할 때, 그리고 기존 요약을 수정할 때 항상 아래 기준을 따른다.

1. 분량 기준 (핵심)

  • 최소 하한: 자막(무공백 기준) 대비 20% (무조건 준수).
  • 목표: 20~50% 범위에서 최대한 충실히. 정보 밀도가 높으면 30% 이상까지.
  • 군더더기 금지이되, 20%에는 반드시 도달시킨다(자막의 실질 내용을 더 상세히 풀어 20%를 채운다).
  • 비율 계산 시 본문에서 마크다운 기호(# > * [] ())는 제외하고 무공백 글자수만 센다(최종 감사 도구 기준과 동일).
  • 자막 유형별 경험칙(단, 모두 20% 이상):
    • 의학·정보 정리형: 25~35%
    • 증권사 시황·대담: 22~30%
    • 러닝/코스/일상 브이로그: 20~22%
    • 1시간+ 뉴스·토크 브리핑·오디오북: 최소 20%(실질 내용을 더 상세히 반영해 확보)
  • 이미 35% 이상이면 수정 불필요(건너뜀).

2. 내용 기준

  • 자막의 실질 정보(수치·인물·직책·논거·종목·기전)는 빠짐없이 반영.
  • 잡담·인사·댓글 낭독·사칭 경고·구령(“하나 둘”)·호흡·풍경 등 부수 내용은 요약에 넣지 않는다.
  • 추정·창작 금지. 자막에 없는 사실을 지어내지 않는다.

3. 검증 절차 (필수)

  1. 요약 작성/수정 후 원자막과 비교한다.
  2. 무공백 비율 = (요약 ## 요약 이후 본문 무공백 글자수, 마크다운 기호 제외) / (자막 무공백 글자수).
  3. 20% 미만이면 자막에서 빠진 실질 내용을 찾아 보완한다(경계값은 여유 있게 20%를 넘긴다).
  4. 더 넣을 실질 내용이 없으면 그 지점을 ‘내용완결’로 보고 마무리(정직한 상한 존중)하되, 하한 20%는 반드시 지킨다.

4. 형식 기준 (2026-07-17 개정: 듣기/TTS 편의를 위해 채널·주의문을 맨 아래로)

  • 문서 순서: ① YAML 프론트매터 → ② # 제목 → ③ ## 요약(본문, ### 소섹션) → ④ 구분선 --- → ⑤ > 채널 | 정리일 | 길이 | [영상 보기] 줄 → ⑥ > ⚠️ 주의 문구.
    • 즉 채널 메타 줄과 주의 문구는 제목 바로 아래가 아니라 요약 본문 다음, 문서 맨 아래에 배치한다(듣기 버튼으로 청취 시 제목 다음 바로 요약이 재생되도록).
  • 단일 일관 본문: ## 요약 아래 ### 소섹션으로 구성(별도 상세 섹션 분리 금지).
  • YAML 프론트매터 항목: title, channel, guest, date, url, tags.
  • 한국어, 간결·정확.
  • 파일명: <yyMMddHHmm>_<제목정리>_<채널>.md, 저장 위치: 이 폴더(youtube 요약).

형식 예시

---
title: "..."
channel: ...
date: ...
url: ...
tags: [youtube요약, ...]
---

# 제목

## 요약

### 소주제 1
...

### 소주제 2
...

---

> 채널: ... | 정리일: ... | 영상 길이: ... | [영상 보기](URL)

> ⚠️ (필요 시) 주의 문구.

5. 워크플로

  • 자막 추출: 사용자 PC에서 python -m yt_dlp(Desktop Commander).
  • 중복 URL(이미 요약된 영상)은 건너뜀.
  • GitHub 게시·텔레그램 발송은 사용자가 “텔레그램에 보내줘 / ww로 보내줘”라고 명시할 때만 수행.
  • 내부 경로(/sessions/…)는 노출하지 않음.
